# 功能状态

本文基于当前仓库当前实现整理，用于帮助用户快速了解哪些功能已经可用、哪些仍在完善、哪些待实现。

| 功能 | 状态 | 说明 |
|:----------------------------------------|:--:|:--------------------------------------------------------------|
| 产品任务 `POST /api/image-tasks/generations` | ✅  | 已支持，用于图片生成，并可通过任务历史返回多张图片。 |
| 产品任务 `POST /api/image-tasks/edits` | ✅  | 已支持，可上传图片进行编辑。 |
| HappyImage-owned 外部兼容入口 | ✅  | 已移除；模型调用由图片任务接口使用当前用户选中的上游供应商 Base URL。 |
| 同时生成多张图片 | ✅  | 已支持，后端与前端都可进行多图生成。 |
| 图片并行生成 | ✅  | 多张图片使用独立线程和账号同时生成，可通过 `image_parallel_generation` 配置关闭。 |
| 图片生成进度追踪 | ✅  | 任务显示排队、运行、完成和失败状态，支持耗时统计。 |
| 图片结果本地化 | ✅  | 网关返回远程 URL 或 b64 图片后，会保存为 Happy Token 私有图片 URL。 |
| 前端图片工作台 | ✅  | 已支持图片生成、图片编辑、模型选择、历史记录与查看大图。 |
| 前端图片懒加载与滚动优化 | ✅  | LazyImage 懒加载、会话切换滚动位置保存与恢复、bfcache 页面恢复同步。 |
| 前端图片输入 / 参考图交互 | ✅  | 已支持参考图上传、预览、移除和编辑模式工作流。 |
| 用户模型供应商 | ✅  | HappyToken 默认供应商自动绑定并默认选中；普通用户可在 Web “我的 -> 供应商”中添加预设或自定义 OpenAI-compatible 供应商。 |
| Cherry Studio 接入 | ✅  | 已支持作为绘图接口接入 Cherry Studio。 |
| New API 接入 | ✅  | 已支持接入 New API。 |
| 本地账号池 / CPA / Sub2API 管理 | ✅ | 已移除，模型渠道与账号池统一交给 NewAPI/OpenAI-compatible 供应商。 |
| Docker 自托管部署 | ✅  | 已支持 Docker Compose 部署，并提供多架构镜像；部署、升级和排障说明见 `docs/docker-deployment.md`。 |
| 兼容接口中的多参考图能力 | ✅  | 已实现，支持在兼容接口中传入多参考图。 |
| 更高级的 Token 调度策略 | ⚠️ | 当前已有基础轮询与限流刷新机制，更复杂的调度策略仍在完善中。 |
| Render / Vercel 等部署表述 | ⚠️ | 当前主要以 Docker 部署为主，其他平台部署方式暂未重点说明。 |
| 图片尺寸参数 | ❌  | 待实现。 |
| 服务端图片 URL 缓存 | ✅  | 已实现。 |
| Anthropic 协议支持 | ❌  | 不在当前后端职责内。 |
